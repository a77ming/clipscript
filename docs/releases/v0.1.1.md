# v0.1.1

## Highlights

- removed stale desktop-packaging metadata that no longer matched the public repository
- upgraded `next` and refreshed production dependencies to clear current production audit findings
- replaced a personal deployment script with an environment-driven deployment template suitable for a public repo
- added explicit maintainer ownership, support scope, and code ownership files for contributors

## Why this release matters

- the repository is easier to evaluate as a maintained open-source project
- contributors now have a clearer path for support, security disclosure, and ownership expectations
- the dependency tree is smaller and cleaner for routine maintenance work
