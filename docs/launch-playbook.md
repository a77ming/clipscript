# Launch Playbook

This file is for turning ClipScript into a repo that can actually pick up stars.

## 1. Repository name

Current repository name:

- `clipscript`

Backup names that also work:

- `subtitle-to-clips`
- `paperedit-ai`
- `reelscript`

## 2. GitHub description

Use this as the repo description:

`Turn subtitle files into short-form highlight clips with AI-generated edit briefs and one-click exports.`

## 3. Suggested GitHub topics

Use 10 topics max:

- `video-editing`
- `subtitle`
- `shorts`
- `tiktok`
- `instagram-reels`
- `ffmpeg`
- `openai`
- `creator-tools`
- `nextjs`
- `ai-tools`

## 4. First release checklist

1. Upload a 15-30 second demo GIF or MP4 to the README.
2. Add 3 screenshots:
   home screen
   clip review screen
   export result screen
3. Create `v0.1.0`.
4. Post the repo the same day on X, Product Hunt, Reddit, and relevant creator communities.

## 5. Post angle

Do not pitch this as a full editor.

Pitch it like this:

> I built a tool that turns subtitle files into short-form clip ideas. Upload an `.srt`, get hooks, captions, voiceover angles, and editor-ready ranges for Shorts, Reels, and TikTok.

## 6. Assets to prepare next

- demo video with one clean source transcript
- one strong before/after example
- README screenshots with English UI
- GitHub social preview after repo rename

## 7. Distribution sequence

Day 1:

- GitHub release
- X thread
- Reddit post in creator / editing / open source communities

Day 2:

- short demo video on X and LinkedIn
- submit to newsletters or AI tool collections

Day 3-7:

- ship one visible improvement per day
- close early issues quickly
- post changelog screenshots

## 8. What gets stars here

The star hook is not “AI”.

The star hook is:

- one-line value proposition
- obvious input and output
- usable demo
- clean README
- fast setup

ClipScript now has the packaging. The next leverage point is a strong demo.
